## Design Context

### Users
- Friend group, ~30 years old, tech-savvy gamers
- Playing UCL Fantasy Football competitively against each other
- 80% desktop, 20% mobile usage
- They want to track rankings, compare performance, and evaluate their fantasy decisions
- Context: casual competitive — friends who take the game seriously but it's still fun

### Brand Personality
**Competitive. Gaming. Bold.**

Emotional goals: competition, gaming pride, evaluation/analysis. The interface should feel like a high-stakes esports tracker — the kind of dashboard that makes you feel powerful when you're winning and motivated when you're not.

### Aesthetic Direction
- **Visual tone:** Dark cyberpunk / esports dashboard
- **Theme:** Dark-only, no light mode
- **Primary accent:** Fire orange (#FF5722) for actions, highlights, active states
- **Secondary accent:** Cyan neon (#00f0ff) for borders, focus, subtle highlights
- **Position colors:** GK gold (#FFB300), DEF green (#4CAF50), MID blue (#42A5F5), FWD red (#EF5350)
- **Colors to AVOID:** Purple, blue as primary — blue is acceptable only for midfielder position coding
- **Background system:** Dark navy layers (#0a0c14 base -> #0f1120 surface -> #141828 card -> #1a1f35 card2)
- **Anti-references:** Nothing over-the-top, nothing childish, no rainbow color explosions, no excessive in-your-face animations
- **References:** Esports dashboards, competitive gaming UIs, dark-themed sports analytics tools

### Typography System
- **Display numbers:** Clash Display 700 (rankings, points, big stats)
- **Navigation/headings:** Barlow Condensed 700-800, uppercase, tight tracking
- **Labels/badges:** Oswald 600-700, uppercase
- **Body text:** DM Sans 400-500, generous line-height (1.5-1.7)
- **Secondary body:** Source Sans 3

### Design Principles

1. **Restrained intensity** — Bold and competitive but never chaotic. Every visual effect must earn its place. Fire animations are reserved for exceptional performance, not decoration. Avoid visual noise.

2. **Layered depth over flat surfaces** — Use the background hierarchy (base -> surface -> card -> floating) consistently. Color-tinted shadows, glassmorphic borders, and subtle noise textures create depth without clutter.

3. **Data-first, decoration-second** — Rankings, points, and player stats are the primary content. Typography hierarchy and spacing should make data scannable at a glance. Chrome and effects support readability, never compete with it.

4. **Position-coded consistency** — GK/DEF/MID/FWD colors are sacred and universal. They appear in badges, shadows, hover states, and card accents. Never use position colors for non-position purposes.

5. **Earned spectacle** — Fire tiers (t1/t2/t3) are progressive reward states. Tier 1 is subtle (green border). Tier 2 adds glow and particles. Tier 3 is full blaze. This escalation must feel meaningful — if everything glows, nothing does.

### Animation Guidelines
- Spring easing: `cubic-bezier(.34,1.56,.64,1)` for playful entrances
- Only animate `transform` and `opacity` — never `transition-all`
- Stagger entrance sequences by row/position (FWD -> MID -> DEF -> GK)
- Fire/particle animations: slow, ambient (1.5s-2.5s), ease-in-out
- Hover interactions: fast (0.15-0.2s), subtle transform + shadow shift
- Keep continuous animations (glows, particles) at low intensity — they run indefinitely

### Constraints
- Single HTML files with inline CSS/JS
- Tailwind via CDN
- Desktop-first (80/20 desktop/mobile split)
- Data served from Vercel serverless backend
