# Design Context — FF Dashboard

## Target Audience
Private group of friends in a UCL Fantasy league. Tech-savvy, competitive, care about stats and bragging rights.

## Use Cases
1. Checking live matchday scores and player performance
2. Viewing live table standings across the league
3. Scouting players for new signings (transfers)

## Brand Personality & Tone
- **Premium** — Apple-level polish, clean surfaces, confident typography
- **Sports data** — Football Manager / fantasy app inspiration (data-dense but organized)
- **Animated** — Purposeful motion that feels alive, not decorative
- **Competitive** — Fire tiers, rankings, and performance should feel intense

## Aesthetic Direction
Dark theme with warm accents (fire orange for top performers, green for clean sheets). Layered depth system. Card-based player UI with tiered visual treatment based on performance. Motion-rich but GPU-friendly (transform + opacity only).

## Constraints
- Single index.html with inline CSS/JS
- Tailwind via CDN
- Must work on desktop primarily (friends check on laptops/phones)
- Data served from Vercel serverless backend
